### SQLite schema + migrations
- Migration system in `src/db/` with version tracking (`schema_migrations`)
- Initial migration creates:
  - `events` — episodic log (type, payload, created_at)
  - `facts` — semantic key-value store
  - `audit_log` — policy evaluation trail (action, decision, reason, metadata)
- WAL journal mode, foreign keys enabled
- Idempotent: re-running migrations on an up-to-date DB is a no-op

### Logging
- `logExternalCall` / `logExternalCallSync` wrap external integrations with input, output, duration
- Redaction layer (`src/lib/redact.ts`):
  - PRIVATE_KEY, MNEMONIC, API_KEY, passphrase/password key names
  - Hex strings ≥ 64 chars
  - Recursive object key redaction

### CLI
- `pnpm dev` — boots agent, runs migrations, logs component readiness
- `pnpm dev --live` — sets live mode flag (execution gates come in M2+)
- `pnpm audit tail [n]` — inspect audit log
