### Rule types (all implemented + tested)
| Rule | Behavior |
|------|----------|
| Kill switch | Deny all actions when flag file exists |
| Allowlist — crypto | Deny if `cryptoAddress` not in list |
| Allowlist — Index categories | Deny if `recipientCategory` not in list (empty list = deny all) |
| Per-tx cap | Deny if `notionalNgn` exceeds cap |
| Category daily cap | Deny if category spend would exceed `category_caps.*.daily_ngn` |
| Daily cap | Deny if global daily total would exceed cap |
| Weekly cap | Deny if global weekly total would exceed cap |
| Velocity | Deny if too many allowed/confirm actions in window |
| Confirm threshold | `confirm` (not deny) when amount ≥ threshold and all other rules pass |

### Decision API
```typescript
engine.evaluate(action) → Result<PolicyEvaluation, PolicyError>
// PolicyEvaluation = { decision: 'allow' | 'deny' | 'confirm', reason: string }
```

### Audit log
- Every successful evaluation persisted via `createPolicyEngineWithAudit`
- Invalid actions return `err` without audit entry
- Metadata stores full action for usage queries (daily/weekly/velocity)

### Kill switch
- Path: `os.tmpdir()/agent.kill` (see `docs/questions.md` for deviation from spec's `/tmp/agent.kill`)
- `pnpm kill` — activates
- `pnpm resume` — requires typing `yes`

### CLI
| Command | Status |
|---------|--------|
| `pnpm policy show` | Prints active policy JSON |
| `pnpm audit tail [n]` | Shows last n audit entries |
| `pnpm kill` | Activates kill switch |
| `pnpm resume` | Deactivates kill switch (with confirmation) |

## Deviations from spec

1. **Kill switch path:** Uses `os.tmpdir()/agent.kill` instead of hardcoded `/tmp/agent.kill` for Windows compatibility. Documented in `docs/questions.md`.

2. **Policy fallback:** Loads `config/policy.example.yaml` when `config/policy.yaml` is missing. Conservative caps still apply; operator should copy example to `policy.yaml` before live use.

3. **Unified `notionalNgn`:** All action kinds use a single NGN notional field for cap math. Crypto-specific USD caps deferred to M3 when LI.FI quotes provide conversion.

## Known issues

- Velocity/daily/weekly usage queries only count audit entries with `decision = 'allow'` (or `allow`+`confirm` for velocity). Denied attempts do not consume velocity budget — intentional (failed attempts shouldn't block legitimate ops).
- `pnpm resume` requires interactive terminal; non-TTY environments will hang.
